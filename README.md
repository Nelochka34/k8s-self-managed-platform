# k8s-self-managed-platform

## Цель проекта: 
Развернуть Kubernetes-платформу, включающую:
- self-managed Kubernetes cluster;
- отказоустойчивую систему управления;
- мониторинг кластера и приложений;
- централизованное логирование;
- CI/CD pipeline для приложения;
- GitOps-подход к развертыванию компонентов платформы.

## MVP

В MVP входят:
- 3 control-plane узла с etcd
- 2 worker узла
- ingress-nginx
- ArgoCD
- Prometheus, Grafana, Alertmanager
- Loki + Vector
- тестовое приложение Online Boutique
- GitHub Actions
Для логирования выбран Vector, так как он может собирать и обрабатывать логи кластера и приложений перед отправкой в Loki.

## План работ

1. Подготовка репозитория и структуры проекта.
2. Подготовка инфраструктуры Kubernetes.
3. Развертывание Kubernetes-кластера.
4. Установка GitOps-инструмента ArgoCD.
5. Установка мониторинга Prometheus/Grafana и Alertmanager.
6. Установка централизованного логирования.
7. Подготовка demo-приложения.
8. Настройка CI/CD pipeline.

## Вывод:
В рамках курсовой работв была спроектирована и реализована self-managed Kubernetes платформа с элементами высокой доступности и автоматизации процессов эксплуатации. 

Для развертывания кластера использовался Kubespray, позволяющий автоматизировать создание  настройку инфраструктуры Kubernetes. 
В результате был создан отказоустойчивый кластер Kubernetes версии 1.36, состоящий из трех узлов управления (control-plane) и двух рабочих узлов (worker), обеспечивающий высокую доступность управляющих компонентов. 

Для организации сетевого взамодействия был настроен Calico в качестве сетевого плагина. Для публикации приложений был установлен ingress-nginx, а для предоставления сервисам внешних IP-адресов использован MetalLB. Внешний доступ к сервисам реализован через собственный домен nelafree.su. 

В качестве тестового приложения была развернута микросервисная платформа Online Boutique, позволившая проверить работоспособность механизмов маршрутищации трафика, мониторинга и централизованного логирования. 

Для реализации GitOps-подхода был внедрен ArgoCD. Все конфигурации инфраструктуры и приложений хранятся в Git-репозитории и автоматически синхронизируются с кластером Kubernetes. 

Для мониторинга платформы был развернут стек Prometheus, Grafana. Настроен сбор метрик с узлов кластера, а также визуализация данных через Grafana. 

Для реализации централизованного логирования был внедрен стек Loki и Vector. Логи контейнеров автоматически собираются со всех узлов кластера, передаются в Loki и доступны для анализа через Grafana. 

Для демонстрации процессов CI/CD было создано приложение demo-app. Автоматизация сборки реализована с использованием GitHub Actions. Развертывание приложения в кластере осуществляется посредством ArgoCD в соответствии с принципами GitOps. 

В результате выполнения работы была создана self-managed Kubernetes-платформа, включающая механизмы высокой доступности, мониторинга, централизованного логирования, GitOps и CI/CD.
